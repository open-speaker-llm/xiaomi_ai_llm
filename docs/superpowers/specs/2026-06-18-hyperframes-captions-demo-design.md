# HyperFrames 字幕与实拍演示设计

## 目标

修正技术名词被跨字幕拆分、下划线被语音读出、字幕句末标点冗余三个问题，并把 94 秒一镜到底演示完整插入现有讲解视频。

## 文本与音频

- `narration.txt` 保留面向观众的技术写法，作为字幕原稿。
- `narration-tts.txt` 使用自然口语表达脚本名、变量名和路径，供 EdgeTTS 合成，避免朗读“下划线”。
- EdgeTTS 输出新的旁白音频和时间戳字幕；字幕生成器再将自然读法映射回屏幕上的技术写法。
- 字幕按中文语义切分，英文单词、变量、路径和脚本名视为不可拆分单元，所有字幕移除句末标点。

## 演示视频

- 在“再把用户问题转给 LLM”之后插入 `/Users/mac-mini-wx/Movies/6月16日(1).mov`，让实拍先证明 fallback 到大模型的实际效果，再继续解释内部判断逻辑。
- 演示视频保持 1920x1080 全屏、完整 94 秒并保留原声。
- 演示期间旁白和讲解字幕暂停；原视频已有的画面与烧录字幕保持不变。
- 演示结束后的第一句改为“这段实测背后，关键不是关键词匹配，而是读取原生结构化结果。”，下一句继续说明 `domain/action`、`speak/to_speak` 和 `query` 的判断顺序。
- 演示结束后恢复旁白，后续场景、字幕和总时长整体顺延。

## 验证

- 单元测试覆盖技术标识不拆分、句末标点移除、显示文本映射和插入点后的时间偏移。
- 运行 HyperFrames lint、validate、inspect。
- 在插入点前、中、后检查画面与声音轨道衔接，并输出预览。
